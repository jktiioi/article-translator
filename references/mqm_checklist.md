# MQM 审校清单（自审 / 双智能体交叉校验用）

基于 MQM（Multidimensional Quality Metrics，已被采纳为 ASTM F3131）。官方核心 7 维；本清单聚焦文章翻译最相关的 6 维。逐条核对，发现即记录 `{issue, severity, dimension, location, suggestion}`。severity：critical（影响理解/事实错误）> major（明显不当）> minor（细微）。

## 1. Accuracy 准确性
- [ ] 无漏译（原文每句/段都有对应）
- [ ] 无误译（语义与原文一致）
- [ ] 无幻觉 / 增译（未编造原文没有的信息）
- [ ] 专名、数字、日期、金额、单位**原样保留**（重点查量级：million/billion ↔ 万/亿）
- [ ] 代码、URL、HTML 标签未被翻译或破坏
- [ ] `[[PROT\d+]]` 占位符全部完好、数量与还原后一致
- [ ] 极性/方向/否定类表述与原文逐句对齐、未被翻转（增减、正负、支持反对、启停、能否等）

## 2. Terminology 术语
- [ ] 全文术语译法一致（跨段/章不漂移）
- [ ] 术语表（glossary）中的词全部按表翻译
- [ ] 同一概念不出现多种译法

## 3. Fluency 流畅度
- [ ] 目标语自然，无翻译腔
- [ ] 无语法错误、无不连贯句式
- [ ] 长句拆分合理，可读性强

## 4. Style 风格
- [ ] 语气/语域符合受众（正式/口语）
- [ ] 目标语人称/敬语/语体规范（如是否需要敬称、T-V 区分）符合习惯
- [ ] 习语、典故处理地道（不过度直译也不硬意译）

## 5. Locale convention 地区格式
- [ ] 日期/数字/货币/地址按目标语地区格式（如 ru-RU 用 DD.MM.YYYY、千位分隔符）
- [ ] 数字小数分隔符与单位本地化按目标语地区格式统一，不混用原语或拉丁缩写
- [ ] 标点与符号（范围连接符、省略号等）全文用法一致，不前后矛盾
- [ ] 注意：本维度是"格式合规"，不是"文化适配"；文化适配见 Style 维度

## 6. Design 版式/排版（文章级）
- [ ] 标题层级（# 数量）与原文一致
- [ ] 列表、表格、引用块结构完整
- [ ] 字体/加粗/斜体标记保留
- [ ] **超链接可点击、href 完整**
- [ ] 超链接/字段显示文本与相邻词、标点间为单一空格：无跨段粘连、无连续双空格（逐链接拼前导/显示/后导文本核对）

## 双智能体用法
- 翻译 Agent 产出译文；反思 Agent（或同模型第二遍）按本清单产出问题 JSON。
- 反思须显式从原文重推语义，核对极性/方向/否定类表述未被翻转；不可仅靠译文自洽性判断。
- critical / major 问题必须修订；自审不确定或可能回归处，标记"需人工复核"而非自动定稿。
- 机械层（渲染态校验）只查确定性规则；语境合法性粘连与语义方向由 AI 语义层/人工复核，脚本硬扫会误伤合法粘连或漏掉语义翻转。
- 关键内容（合同/医疗/法律/财务）无论评分，均提示用户人工终审。
