# 文章翻译（大模型驱动 · 保排版 + 链接 + 审校）

## 原则（本 skill 的设计立场）
- **语言是参数，不是示例**。源/目标语言、术语表由调用时提供；skill 不内置任何语言的示例词条，新增语言只需补一份术语表，工作流不变。
- **只编码方法论（原则），不枚举示例**。翻译质量靠"保护非译元素 + 三步法 + 术语约束 + 结构化审校"这套可复用流程保证，而非堆示例。
- **非译元素靠机制保护，不靠模型自觉**。链接/代码/HTML/公式/占位符在译前抽出，译后还原并校验，模型始终看不到原始非译内容。
- **自审既要润色，也要兜底准确性，且须聚焦方向而非仅通顺**。反思对流畅/一致有帮助，但对准确性非稳定增益，可能回归；须显式从原文重推语义，核对极性/方向/否定类表述未被翻转，再辅以术语表、规则校验、人工/强模型复核兜底。
- **AI 出稿是草稿**。关键内容（合同/医疗/法律/财务）无论评分都须人工终审。
- **还原后校验看渲染态，但机械校验有确定边界**。译文按 run/段切分，跨 run 与超链接字段边界的空格、数字/单位格式、符号用法易在逐段翻译中漂移或丢失；占位符齐全不等于成品合格，还原进 docx 后必须基于段落拼接文本再做一轮机械校验。
- **机械校验与语义校验分工明确（分层校验）**。脚本只负责可枚举的确定性规则——占位符计数、连续双空格、CJK/原语残留、URL 完好、数字/符号格式枚举是否一致；它判断不了"该粘还是该分"的语境合法性，更判断不了语义方向。这两类必须交给 AI 语义层或人工复核，不能用脚本硬扫后机械定稿，否则会误伤合法粘连或漏掉语义翻转。

## 这个 skill 解决什么
用户要翻译长篇文章/博客/文档/网页正文（Markdown / 纯文本 / Word .docx）到另一语言，要求保留排版、保留超链接、译后审校、语言可配置。
它不是调用 DeepL/Google/Yandex 等外部引擎 API，而是大模型走一套工程化翻译流程。要最高专业质量，可先引擎打底、再走本流程润色。

## 好翻译的标准（审校依据）
采用 **MQM（Multidimensional Quality Metrics，已被采纳为 ASTM F3131）** 框架。官方核心为 7 维；本 skill 聚焦与文章翻译最相关的 6 维（完整清单见 `references/mqm_checklist.md`）：

| 维度 | 含义 | 说明 |
|---|---|---|
| Accuracy 准确性 | 无漏译/误译/幻觉 | 专名、数字、日期、单位原样保留 |
| Terminology 术语 | 术语一致 | 全文同一概念译法统一 |
| Fluency 流畅度 | 目标语自然 | 无翻译腔、无语法硬伤 |
| Style 风格 | 语气/语域/文化适配 | 正式/口语、敬语、习语地道处理 |
| Locale convention 地区格式 | 日期/数字/货币/地址格式合规 | 如 ru-RU 用 DD.MM.YYYY；注意：这是"格式合规"不是"文化适配" |
| Design 版式/排版 | 标题/列表/表格/链接结构保真 | 对应社交反馈最高频痛点 |

社交平台共识：排版不乱 + 链接在 + 能对照原文 是基础体验；准确优先于流畅；AI 出稿只是草稿，关键内容要人工终审。

## 核心方法（原则，非示例）
1. **保护非译元素**：译前脚本把链接、代码块、行内代码、图片、HTML 标签、裸 URL、引用式链接抽出为 `[[PROT1]]` 占位符，store 存原值；译后还原，并校验每个占位符恰好出现一次。
2. **三步法**：初译 → 反思 → 改进（可迭代）。反思须覆盖六维，且对准确性须显式从原文重推语义、核对极性/方向/否定类表述未被翻转，据问题修订；准确性问题须术语表/规则/人工复核兜底，不可仅靠译文自洽性判断。
3. **术语约束**：用户提供术语表（CSV：原文,译文）即用；未提供则先抽取关键术语建表，全程注入约束。
4. **结构化审校**：按 MQM 维度逐条自查，输出 JSON `{issue, severity, dimension, location, suggestion}`；critical/major 必须修订，不确定处标记人工复核。
5. **双语对照**：默认输出原文∥译文便于人工核对。

## 工作流

### 第 0 步 准备
确认：源语言 / 目标语言 / 受众与语气 / 领域 / 是否双语对照。术语表：用户给 CSV 即用；否则让模型从原文抽 20–50 个关键术语建表（模板 `references/glossary_template.csv`）。

### 第 1 步 保护非译元素
按输入类型选择抽取器（两者都把非译元素护成占位符，译后还原）：
- **Markdown / 纯文本**：
  ```bash
  python scripts/protect_links.py protect --in <raw.md> --out <protected.md> --store <store.json>
  ```
  脚本护住 链接/代码块/行内代码/图片/HTML href/src/裸URL/引用式链接。
- **Word .docx**（依赖 python-docx，已随 venv 提供）：
  ```bash
  python scripts/protect_docx.py extract --in <raw.docx> --out <src.txt> --store <store.json>
  ```
  脚本遍历正文/表格/页眉页脚/脚注的所有 run；**超链接 URL 存于 document.xml.rels，天然不进译文**，仅译显示文本并原地回填（排版/样式/表格保真）；正文裸 URL 抽成 `[[PROT#]]` 护住；跳过域代码（目录 TOC/页码/交叉引用）。

### 第 2 步 初译
按 `references/prompt_templates.md` 初译 prompt 翻译 protected.md。要点：system 锁定语气/受众/领域；严格保留 `[[PROT\d+]]`；长文按标题分节，每节带术语表与前文已定译名，并以上一段落作上下文。

### 第 3 步 反思
按反思 prompt，对照 `references/mqm_checklist.md` 逐条自查，输出问题 JSON。

### 第 4 步 改进
据反思结果修订初译。准确性问题须术语表/规则复核兜底；关键内容提示人工终审。

### 第 5 步 还原 + 校验 + 输出
- **Markdown / 纯文本**：
  ```bash
  python scripts/protect_links.py restore --in <translated.md> --store <store.json> --out <final.md>
  ```
- **Word .docx**：
  ```bash
  python scripts/protect_docx.py restore --docx <raw.docx> --in <translated.txt> --store <store.json> --out <final.docx>
  ```
  还原前校验每个 `[[DOCX#]]` 恰好一次；还原后检查无 `[[PROT#]]` 残留。默认双语对照，单语可只给译文。
  **还原后渲染态校验（必做，机械层）**：占位符齐全不等于成品合格。基于段落拼接文本再做一轮机械校验——① 超链接/字段显示文本与相邻词、标点保持单一空格，无跨段粘连、无连续双空格（仅查确定性空格规则，不判定语境合法性）；② 数字小数分隔符与单位按目标语地区格式统一，不混用原语/拉丁缩写；③ 标点与符号（范围连接符、省略号等）全文用法一致。**机械层不覆盖语义方向**：极性/方向/否定是否翻转、以及"该粘还是该分"的语境合法性，须由 AI 语义层（第 6 步）或人工复核，脚本硬扫会误伤合法粘连。

### 第 6 步 语义核验（AI 层，与机械层互补）
机械层只查确定性规则，本步对还原后的成品做一轮 AI 语义核验，覆盖机械层的两类盲区：① 极性/方向/否定类表述与原文逐句对齐，确认无翻转；② 跨 run/跨链接处的粘连，判断是否语境合法（该粘的分不应误拆）；③ 不确定处标记"需人工复核"而非自动定稿。关键内容无论本步评分均提示人工终审。

## 语言配置（参数化）
- 默认支持 EN↔ZH、EN↔RU；加语言只改 prompt 的源/目标语言名 + 提供该语言术语表，工作流不变。
- 目标语特有的形态/语法一致性要求（如格、数、性、敬称、语体）因语言而异，由术语表与该语言审校项覆盖，不在 skill 内写死示例。

## 边界与提醒
- 数字、日期、金额、单位：模型可能漂移，量级（million/billion ↔ 万/亿）尤易错，必须逐条核验。
- 极性/方向/否定类表述：模型在流畅度压力下易翻转，须逐句从原文重推语义核验，不能只看译文是否自洽。
- 超长文：分节翻译并维护运行中术语表，避免前后译名不一；每节带上一段落作上下文。
- 本 skill 是大模型翻译流程，不是外部引擎 API。
- **docx 专用提醒**：① 超链接 URL 在 rels 里，天然不被译乱，显示文本照常译；② 域（目录 TOC、PAGE 页码、REF 交叉引用等）整体跳过不译，含其显示结果；译后如需刷新目录，在 Word 里右键"更新域"即可；③ 代码块（等宽样式文本）未被单独护住，靠 prompt "不译代码" 指令，如需绝对保真可先转 md；④ 复杂艺术字/文本框一般能被通用递归覆盖，建议译后核对。

- `references/prompt_templates.md` — 初译/反思/改进 prompt 模板。
- `references/mqm_checklist.md` — MQM 六维审校清单。
- `references/glossary_template.csv` — 术语表模板（语言无关，含 1 行格式示例）。
